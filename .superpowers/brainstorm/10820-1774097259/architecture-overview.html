Full-stack rewrite: React 19 + Vite → RedwoodJS + PostgreSQL + dbAuth

System Architecture

5-Layer Architecture
🖥️ CLIENT (Browser)
PRESENTATION
Pages • Cells • Components • Layouts
WriterPage → ActiveDocumentCell → Typewriter
UnifiedSyntaxPanel • ThemeCustomizer • Toolbar
↕
STATE
React Context • Custom Hooks • Web Workers
WriterContext → useAppHotkeys, useSyntaxWorker
useFocusNavigation • useCustomTheme • useHarmonicaDrag
↕
STORAGE (Local)
localStorage Write-Ahead Cache • Service Worker
28 localStorage keys → instant writes
SyncManager queue → coalesces & flushes
⚡ SERVER (RedwoodJS API)
DATA (GraphQL API)
SDL • Services • Resolvers • @requireAuth
documents.sdl.ts → documents.ts service
userSettings.sdl.ts → userSettings.ts
sync.sdl.ts → syncBatch mutation
↕
SYNC
Conflict Resolution • Version Control • Migration
Document: server-wins + local snapshot stash
Settings: last-write-wins by clientUpdatedAt
↕
STORAGE (PostgreSQL)
Prisma ORM • Railway Managed DB
User • Document • DocumentSnapshot
UserSettings • CustomTheme • ThemeConfig
← GraphQL over HTTPS + Cookie Auth (dbAuth) →

Data Models (Prisma)

User
id, email, hashedPassword
salt, resetToken
→ has one UserSettings
→ has many Documents
→ has many CustomThemes
Document
id, title, content (Text)
version, checksum
wordCount, charCount
→ has many Snapshots
→ tracked by SyncQueue
UserSettings
themeId, fontId, fontSizeOffset
lineHeight, letterSpacing
highlightConfig (JSON)
focusMode, soloMode
themeOverrides (JSON)
1 row per user
CustomTheme
name, all color fields
highlightColors (JSON)
rhymeColors (JSON)
max 20 per user
ThemeConfig
hiddenThemeIds (JSON)
themeOrder (JSON)
hasCustomizedVisibility
1 row per user
DocumentSnapshot
content, version, checksum
wordCount, createdAt
max 50 per document
auto-pruned oldest
